## 🛠️ 项目 A：学习计划 CLI 工具

### 项目概述

创建一个命令行工具，用于管理和跟踪学习进度。

### 功能需求

1. **课程管理**
   - 列出所有课程
   - 显示课程详情
   - 搜索课程内容
   - 按主题筛选

2. **进度跟踪**
   - 标记课程完成
   - 显示学习进度
   - 计算完成百分比
   - 生成进度报告

3. **统计功能**
   - 学习时长统计
   - 完成课程数量
   - 学习路径建议
   - 下一步推荐

4. **数据管理**
   - 导出学习记录
   - 导入学习记录
   - 数据备份
   - 数据恢复

### 实现步骤

#### 第一步：规划

```
进入 Plan 模式，规划 CLI 工具：

1. 技术选型
   - Node.js + Commander.js
   - 或 Python + Click
   - 或 Rust + Clap

2. 命令设计
   - learn list
   - learn show <id>
   - learn complete <id>
   - learn progress
   - learn export

3. 数据结构
   - 课程数据格式
   - 进度数据格式
   - 存储方案

让 Claude 创建完整计划。
```

#### 第二步：骨架搭建

```
创建项目结构：

cli-tool/
├── src/
│   ├── commands/      # 命令实现
│   ├── utils/         # 工具函数
│   ├── data/          # 数据管理
│   └── index.js       # 入口文件
├── tests/             # 测试文件
├── docs/              # 文档
└── package.json

使用 Claude Code 生成初始结构。
```

#### 第三步：核心功能

```
实现核心命令：

1. 数据加载器
   - 读取课程文件
   - 解析 Markdown
   - 提取元数据

2. 命令实现
   - list 命令
   - show 命令
   - complete 命令
   - progress 命令

3. 数据持久化
   - 保存进度
   - 加载进度
   - 数据验证

逐步实现并测试。
```

#### 第四步：增强功能

```
添加高级功能：

1. 搜索功能
   - 全文搜索
   - 关键词高亮
   - 相关推荐

2. 报告生成
   - Markdown 报告
   - JSON 导出
   - 可视化图表

3. 交互界面
   - 进度条
   - 彩色输出
   - 确认提示

让 Claude Code 协助实现。
```

#### 第五步：测试和文档

```
完善项目：

1. 单元测试
   - 命令测试
   - 工具函数测试
   - 数据处理测试

2. 集成测试
   - 端到端测试
   - 用户场景测试

3. 文档编写
   - README.md
   - 使用指南
   - API 文档
   - 贡献指南

4. 打包发布
   - npm package
   - 安装脚本
   - 示例配置
```

### 检查清单

- [ ] 可以列出所有课程

## 🔧 项目 B：练习验证系统

### 项目概述

为学习计划添加自动练习验证系统。

### 功能需求

1. **练习检测**
   - 识别代码练习
   - 提取练习要求
   - 分析练习类型

2. **自动验证**
   - 语法检查
   - 功能测试
   - 代码质量检查
   - 最佳实践验证

3. **反馈系统**
   - 验证结果
   - 改进建议
   - 学习提示
   - 示例代码

4. **进度管理**
   - 练习状态
   - 完成度追踪
   - 成就系统

### 实现步骤

#### 第一步：分析现有代码

```
使用 Explore 代理分析：

1. 找出所有练习
2. 分析练习模式
3. 识别验证点
4. 设计验证方案

让 Explore 代理完成深度分析。
```

#### 第二步：设计验证框架

```
进入 Plan 模式，规划验证系统：

1. 验证架构
   - 验证器接口
   - 练习类型
   - 验证策略

2. 数据模型
   - 练习元数据
   - 验证结果
   - 反馈格式

3. 扩展机制
   - 自定义验证器
   - 插件系统

创建详细实现计划。
```

#### 第三步：实现核心验证器

```
实现验证功能：

1. JavaScript 验证器
   - ESLint 集成
   - 单元测试
   - 代码覆盖

2. Python 验证器
   - Pylint 集成
   - Pytest 测试
   - 类型检查

3. 通用验证器
   - 文件存在性
   - 格式验证
   - 内容检查

逐步实现各类验证器。
```

#### 第四步：集成和测试

```
集成到学习计划：

1. 练习标记
   ```markdown
   <!-- exercise
   id: ex-001
   type: javascript
   difficulty: easy
   requirements:
     - 使用函数
     - 处理边界情况
     - 添加注释
   -->
   ```

2. 验证命令
   - verify exercise <id>
   - verify all
   - verify status

3. 反馈系统
   - 通过/失败
   - 详细报告
   - 改进建议
```

### 检查清单

- [ ] 可以自动识别练习
- [ ] 可以验证练习完成情况
- [ ] 提供有用的反馈
- [ ] 支持多种语言
- [ ] 易于扩展
- [ ] 有完整的文档

### 项目概述

将学习计划重构为模块化、可扩展的架构。

### 重构目标

1. **模块化结构**
   - 清晰的模块边界
   - 松耦合设计
   - 高内聚组织

2. **可扩展性**
   - 插件系统
   - 主题支持

3. **可维护性**
   - 统一代码风格
   - 完整的类型定义
   - 自动化测试

4. **性能优化**
   - 快速加载
   - 缓存机制
   - 增量处理

### 实现步骤

#### 第一步：现状分析

```
深入分析现有结构：

1. 使用 Explore 代理（very thorough）
   - 分析所有文件

   - 找出耦合点

2. 评估问题
   - 结构混乱
   - 重复代码
   - 缺少抽象

3. 设计新架构
   - 分层设计
   - 模块划分
   - 接口定义

#### 第二步：架构设计

```
进入 Plan 模式，设计新架构：

1. 核心模块
   - 课程管理器

阶段 3：扩展性
- 添加插件系统
- 实现主题支持

- 性能优化
- 缓存实现

## 💡 项目技巧

### 时间管理

```
如果时间有限：

1. 优先实现核心功能
2. 简化非关键功能
3. 减少测试范围
4. 简化文档

确保项目可用，不追求完美。
```

### 遇到困难

```
遇到问题时：

1. 使用 MCP 工具搜索解决方案
2. 让 Claude Code 解释代码
3. 简化实现方案
4. 分解为小步骤

不要放弃，寻求帮助！
```

### 质量保证

```

持续检查：

1. 每完成一个功能，立即测试
2. 定期运行所有测试
3. 检查代码风格
4. 验证文档更新

保持代码健康。
```
