## 🗣️ 第一步：基础对话技巧

### 提问的艺术

好的提问能获得更好的答案：

**❌ 不好的提问：**
```
这段代码有问题
```

**✅ 好的提问：**
```
请帮我检查 exercises/sample-code.js 文件中的代码，
看看是否有潜在的性能问题或 bug。
```

### 上下文的重要性

Claude Code 能记住整个对话历史，所以你可以：

```
# 第一次请求
创建一个函数来计算数组的平均值

# 后续请求（Claude 记得之前的上下文）
现在修改这个函数，处理空数组的情况
```

## 📖 第二步：Read 工具 - 读取文件

Read 工具是最常用的工具之一，用于读取文件内容。

### 基本用法

**直接让 Claude 读取文件：**

```
请读取 exercises/sample-code.js 文件
```

**指定读取范围：**

```
读取 exercises/sample-code.js 的第 10-30 行
```

### 练习 1: 读取文件

```
请读取 exercises/sample-code.js 文件，并告诉我：
1. 这个文件的主要功能是什么？
2. 使用了哪些编程概念？
3. 代码结构如何组织？
```

## 🔍 第三步：Glob 工具 - 查找文件

Glob 工具使用模式匹配来查找文件。

### 常用模式

```
# 查找所有 JavaScript 文件
**/*.js

# 查找特定目录下的文件
exercises/*.py

# 查找多种类型的文件
**/*.{js,ts,jsx,tsx}

# 查找测试文件
**/*.test.js
```

### 练习 2: 文件查找

```
请使用 Glob 工具：
1. 找出所有的 JavaScript 文件
2. 找出所有的 Python 文件
3. 找出 exercises 目录下的所有文件
4. 找出所有 Markdown 文档
```

## 🔎 第四步：Grep 工具 - 搜索内容

Grep 工具用于在文件中搜索特定的文本或模式。

### 基本用法

**搜索简单文本：**

```
在 exercises 目录下搜索 "function" 关键字
```

**使用正则表达式：**

```
搜索所有包含 "export.*function" 的行
```

**限定文件类型：**

```
在所有 .js 文件中搜索 "TODO" 注释
```

### 练习 3: 内容搜索

```
请使用 Grep 工具：
1. 在所有文件中搜索 "TODO" 关键字
2. 在 .js 文件中搜索 "function" 定义
3. 搜索包含 "import" 或 "require" 的行
4. 搜索所有注释行（以 // 开头）
```

## 💡 最佳实践

1. **先查找，后读取**：使用 Glob 定位文件，再用 Read 读取
2. **精准搜索**：使用 Grep 的文件类型过滤提高效率
3. **明确请求**：告诉 Claude 你想了解什么，而不是泛泛地说"查看文件"

## 🔧 工具对比

| 工具 | 用途 | 使用场景 |
|------|------|----------|
| Read | 读取文件内容 | 需要查看完整文件内容时 |
| Glob | 查找文件路径 | 需要定位特定文件或文件类型时 |
| Grep | 搜索文件内容 | 需要查找特定代码或文本时 |

## ⚠️ 常见错误

**错误：尝试用 Grep 查找文件名**
```
❌ 搜索名为 "config.js" 的文件
✅ 使用 Glob 查找 "**/config.js"
```

**错误：一次性读取太多文件**
```
❌ 读取项目中的所有文件
✅ 先用 Glob 定位，再选择性地读取
```
